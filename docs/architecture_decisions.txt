Architecture Decisions

It was decided to choose 3 types of message passing to utilize their advantages for the following use cases:

1. RESTful API: For integrating with Frontend web application. REST is nowadays mostly used industry standard, using Starter project it was simple to integrate.

2. gRPC: For integrating with mobile application to receive device location, because of its overall better performance, payload size reduction and strict typing compared to REST.

3. Kafka: For message wueue of location data so we can guarantee the system can handle large amount of data without overloading.
